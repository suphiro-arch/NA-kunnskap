# Produkt-canvas: data.norge.no

Målgruppe: Hovedfokus er forretningssiden og strategisk arkitektur.

## Navn
data.norge.no

## Ressurs ID
16 (Produktliste NA-kunnskap).

## Status/Livsfase
Produksjon som nasjonal portal for datadeling i offentlig sektor.

## Modenhet
Høy:
- Etablert nasjonal inngang for metadata om datasett, API-er, begreper og informasjonsmodeller.
- Bredt brukt i offentlig datadeling og arkitekturarbeid.
- Integrert med Felles datakatalog-komponenter.

## Kort beskrivelse
data.norge.no er den nasjonale portalen for å oppdage, beskrive og gjenbruke dataressurser i offentlig sektor. Portalen samler metadata og gir oversikt over datasett, API-er, begreper og andre informasjonsressurser som støtter interoperabilitet og datadrevet tjenesteutvikling.

## Kapabiliteter
- Informasjonsforvaltning: Oversikt over datasett
- Informasjonsforvaltning: Oversikt over API
- Informasjonsforvaltning: Oversikt over begreper
- Informasjonsforvaltning: Informasjonsarkitektur
- Datautveksling og integrasjon: Dele data med andre
- Datautveksling og integrasjon: Bruke data fra andre
- Standardisering: Forvaltningsstandarder

Grunnlag: kapabiliteter mappet mot `index/capabilities.yaml`.

## Produktmål
- Øke oppdagbarhet og gjenbruk av offentlige dataressurser.
- Redusere tid fra databehov til faktisk integrasjon/bruk.
- Styrke semantisk og teknisk samhandling på tvers av virksomheter.
- Understøtte nasjonale mål om datadeling og datadrevet innovasjon.

## Brukerbehov
- Virksomheter trenger en felles portal for å publisere og finne dataressurser.
- Utviklere og arkitekter trenger rask tilgang til metadata og koblinger til dokumentasjon.
- Informasjonsforvaltere trenger et nasjonalt rammeverk for standardisert beskrivelse.

## Hvem er brukerne og brukersegmentene
- Offentlige virksomheter som publiserer metadata.
- Utviklere, integrasjonsteam og arkitekter i offentlig og privat sektor.
- Analytikere, forskningsmiljøer og innovasjonsaktører.
- Informasjonsforvaltere og datastyringsmiljøer.

## Hovedfunksjoner
- Søkbart register over datasett, API-er og begreper.
- Visning av metadata med kobling til tekniske og faglige kilder.
- Tilrettelegging for gjenbruk og sammenstilling av informasjon på tvers.
- Støtte for standardisert publisering og metadataforvaltning.

### Scope og avgrensning
- Inngår:
  - metadata- og oppdagbarhetsfunksjoner
  - nasjonal oversikt over datadelingselementer
- Inngår ikke:
  - selve kjøringen av API-er eller dataplattformer hos virksomhetene
  - full intern datastyring i hver enkelt virksomhet

## Veikart over kommende funksjonalitet
- Forventet løpende forbedring av metadata-kvalitet, søkbarhet og brukervennlighet.
- Kontinuerlig utvikling i samspill med Felles datakatalog og nasjonale datadelingsinitiativ.
- Konkrete prioriteringer må avstemmes med Digdirs offentlige roadmap/forvaltningsinformasjon.

## Forretningsverdi/Verdiforslag
- Reduserer dobbeltarbeid ved at eksisterende dataressurser blir synlige.
- Øker tempo i tjenesteutvikling og analysearbeid.
- Bedrer beslutningsgrunnlag for gjenbruk før nyutvikling.
- Styrker interoperabilitet gjennom felles metadata-praksis.

## Utfordringer og risiko
- Juridisk risiko: ufullstendig informasjon om rettigheter/bruksvilkår kan gi feilbruk.
- Teknisk risiko: varierende metadata-kvalitet reduserer gjenfinningsverdi.
- Sikkerhetsrisiko: feil klassifisering av dataressurser kan gi uønsket eksponering.
- Leverandørrisiko: ulik modenhet hos publiserende virksomheter.
- Brukerrisiko: mangelfull datakontekst kan gi feil tolkning av data.
